# Руководство по FlowMVI

## Почему проект использует FlowMVI

- Общему рантайму feature-слоя понадобились более сильные настройки по умолчанию для обработки ошибок, повторных попыток, логирования и lifecycle-teardown.
- Модули FlowMVI доступны через `gradle/libs.versions.toml` и подключены в `kmp/feature/base`.
- `FeatureStoreComponent` служит совместимой точкой входа для feature-компонентов.

## Базовые элементы рантайма

- `FeatureStoreComponent` в `kmp/feature/base` предоставляет состояние, поток действий и безопасный helper для запуска.
- `RecoverPlugin` преобразует исключения в обновления состояния.
- `LoggingPlugin` логирует события действий и ошибок.
- `RetryPlugin` и `SavedStatePlugin` остаются точками расширения для следующего этапа внедрения.

## Чеклист реализации фичи

1. Переведите базовый класс компонента на `FeatureStoreComponent`.
2. Публикуйте действия через `publishAction`.
3. Оборачивайте корутины с побочными эффектами в `launchSafely`.
4. Добавляйте `RecoverPlugin`, чтобы ошибки были видны в UI.
5. При необходимости добавляйте bridge-аннотации для web-экспорта.
